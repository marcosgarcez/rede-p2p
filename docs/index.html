⚙️ Módulo — Grafo

Grafo *criar_grafo(int n)
Aloca e inicializa o grafo com n nós vazios na memória.
▶

Usa malloc para alocar a estrutura principal do grafo, o array de ponteiros para os nós, e cada No individualmente. Cada nó começa com sua lista de vizinhos apontando para NULL (sem conexões). Caso qualquer alocação falhe, o programa encerra com perror e exit(1).

Parâmetros
NomeTipoDescrição
nintNúmero de nós (computadores) que o grafo terá
Retorno
TipoDescrição
Grafo *Ponteiro para o grafo alocado e inicializado
c 
Grafo *criar_grafo(int n) {
    Grafo *g = malloc(sizeof(Grafo));
    g->total = n;
    g->nos   = malloc(n * sizeof(No *));
    for (int i = 0; i < n; i++) {
        g->nos[i]           = malloc(sizeof(No));
        g->nos[i]->id       = i;
        g->nos[i]->vizinhos = NULL;
    }
    return g;
}
void adicionar_aresta(Grafo *g, int a, int b, int lat)
Cria uma aresta bidirecional (não-dirigida) entre dois nós com latência definida.
▶

Como o grafo modela uma rede onde a comunicação é de mão dupla (A fala com B e B fala com A), esta função chama a auxiliar estática inserir_aresta_dir duas vezes: uma para a direção A→B e outra para B→A.

A inserção é feita no início da lista encadeada de vizinhos do nó, sendo a operação de menor custo (O(1)) para uma lista encadeada.

aintID do primeiro nó
bintID do segundo nó
latintLatência da conexão em milissegundos
⏱ Tempo: O(1)
void remover_aresta(Grafo *g, int a, int b)
Remove a conexão entre dois nós, simulando a queda de um link de rede.
▶

Usa um ponteiro para ponteiro (Aresta **atual) para percorrer a lista encadeada e remover o nó sem precisar de uma variável anterior separada. Isso é um truque clássico e elegante em C.

Quando o nó alvo é encontrado, o ponteiro que apontava para ele é atualizado para apontar para o próximo (*atual = (*atual)->proxima), "costurando" a lista, e então a memória do nó removido é liberada com free.

c
/* Técnica do ponteiro para ponteiro */
Aresta **atual = &g->nos[a]->vizinhos;
while (*atual) {
    if ((*atual)->destino == b) {
        Aresta *tmp = *atual;
        *atual = (*atual)->proxima; // costura a lista
        free(tmp);
        return;
    }
    atual = &(*atual)->proxima;
}
⏱ Tempo: O(k)
📦 Espaço: O(1)

Onde k é o grau do nó (número de vizinhos diretos).

void imprimir_grafo(Grafo *g)
Exibe a topologia completa da rede no terminal com cores ANSI.
▶

Itera por todos os nós do grafo (loop for externo) e, para cada nó, percorre a lista encadeada de arestas (loop while interno), imprimindo cada vizinho e sua latência. Exibe (sem conexões) para nós isolados. O separador ──▶ é só adicionado entre vizinhos (nunca no final da linha) através de um if (a->proxima).

void liberar_grafo(Grafo *g)
Libera toda a memória alocada dinamicamente pelo grafo, prevenindo memory leaks.
▶

Para cada nó do grafo, percorre sua lista encadeada de arestas liberando cada Aresta individualmente. Em seguida, libera o próprio nó. Por último, libera o array de ponteiros e a struct Grafo. A ordem correta é de dentro para fora: aresta → nó → array → grafo.

⚠️

Não liberar a memória causaria memory leak. Como o programa roda em loop e pode criar estruturas auxiliares, a liberação correta é essencial para a saúde do processo.

📬 Módulo — Fila (FIFO)

Implementação manual de uma fila por array estático, utilizada internamente pela BFS para controlar a ordem de processamento dos nós. Segue a política FIFO (First-In, First-Out).

enfileirar / desenfileirar / fila_vazia
Operações básicas da fila manual: inserir no fim, retirar do início e checar se está vazia.
▶

A fila usa dois índices (inicio e fim) sobre um array fixo. enfileirar escreve na posição fim e avança o índice. desenfileirar lê da posição inicio e avança o índice. Quando inicio >= fim, a fila está vazia.

c
void enfileirar(Fila *f, int id)  { f->dados[f->fim++] = id; }
int  desenfileirar(Fila *f)       { return f->dados[f->inicio++]; }
int  fila_vazia(Fila *f)          { return f->inicio >= f->fim; }
⏱ Todas: O(1)

📡 Módulo — BFS (Busca em Largura)

void bfs(Grafo *g, int src, int pai[], int prof[], Mensagem *msg)
Broadcast em largura a partir de um nó de origem, priorizando links de menor latência.
▶

Esta é a implementação central do simulador. Diferente de uma BFS tradicional (que expande vizinhos na ordem em que os encontra), esta versão usa uma etapa de ordenação por Insertion Sort antes de enfileirar os vizinhos de cada nó, garantindo que os links mais rápidos sejam processados primeiro.

1
Inicialização

Marca o nó fonte como visitado, define seu predecessor como -1 e profundidade como 0. Enfileira-o.

2
Loop principal

Enquanto a fila não estiver vazia, retira o próximo nó (atual) e imprime as informações de recebimento com uma pausa de 300ms (animação).

3
Coleta de candidatos

Percorre os vizinhos de atual e coleta os ainda não visitados em arrays temporários (cand[], clat[]).

4
Insertion Sort

Ordena os candidatos por latência crescente. Para listas pequenas (poucos vizinhos), é o algoritmo mais eficiente na prática.

5
Enfileiramento

Enfileira os candidatos já ordenados, registrando o pai[] e prof[] de cada um para montar a árvore de predecessores.

Parâmetros
NomeTipoDescrição
gGrafo *O grafo sobre o qual executar a busca
srcintID do nó fonte (origem do broadcast)
pai[]int[]Array de saída: quem enviou a mensagem para cada nó
prof[]int[]Array de saída: número de saltos até cada nó
msgMensagem *Pacote de dados a ser transmitido (pode ser NULL)
⏱ Tempo: O(V + E)
📦 Espaço: O(V)

🌿 Módulo — DFS (Busca em Profundidade)

void dfs(Grafo *g, int id, int visitado[], int pai[], int prof[], int profAtual, Mensagem *msg)
Broadcast em profundidade de forma recursiva, mergulhando fundo em cada ramo antes de explorar o próximo.
▶

Implementada de forma recursiva. Cada chamada representa a "visita" a um nó. A indentação visual no terminal é proporcional à profundidade (profAtual), criando uma saída que se parece com a estrutura da árvore de exploração.

Para cada vizinho ainda não visitado, a DFS o marca como visitado, registra seu pai e profundidade, aguarda 300ms e se chama recursivamente, mergulhando pelo caminho até que não haja mais caminhos a explorar naquela direção. Só então retorna e explora o próximo vizinho.

ℹ️

BFS vs DFS no contexto P2P: A BFS garante que a mensagem chegue ao maior número de nós no menor número de saltos possível. A DFS pode ser mais lenta para atingir nós distantes, mas percorre todos os ramos do grafo de forma determinística.

Parâmetros
NomeTipoDescrição
gGrafo *O grafo sobre o qual executar a busca
idintID do nó sendo visitado na chamada atual
visitado[]int[]Array de controle para não revisitar nós
pai[]int[]Array de saída: predecessores de cada nó
prof[]int[]Array de saída: profundidade de cada nó
profAtualintNível de profundidade da chamada recursiva atual
msgMensagem *Pacote de dados transmitido (pode ser NULL)
⏱ Tempo: O(V + E)
📦 Pilha: O(V)

🌳 Módulo — Árvore de predecessores

void imprimir_predecessores(int src, int pai[], int prof[], int n)
Exibe a tabela de predecessores: quem enviou a mensagem para cada nó e em quantos saltos.
▶

Itera por todos os nós e, para cada um que foi alcançado (prof[i] >= 0), imprime uma linha formatada mostrando: o ID do nó, quem foi seu predecessor (ou -- (fonte) se for a raiz) e o número de saltos.

void imprimir_arvore(Grafo *g, int src, int pai[], int prof[])
Desenha visualmente a árvore geradora de transmissão com símbolos ├── e └──, igual ao comando tree do Linux.
▶

Chama a função estática auxiliar imprimir_sub, que é recursiva. A cada nível da recursão, ela:

1. Coleta os filhos do nó atual (aqueles cujo pai[i] == raiz e prof[i] == prof[raiz] + 1).

2. Para cada filho, imprime o prefixo acumulado de espaçamento e o galho (├── ou └── para o último filho).

3. Calcula o novo prefixo para a próxima chamada: se o filho não for o último, adiciona │ (cano reto) para que os filhos dos próximos irmãos sejam conectados visualmente; se for o último, adiciona espaços em branco.

│   ├── P3  (salto 2, 8ms)
│   │   └── P5  (salto 3, 15ms)
│   └── P4  (salto 2, 30ms)

└── P2  (salto 1, 45ms)

🔍 Módulo — Union-Find

O Union-Find (Conjuntos Disjuntos) é usado para detectar quais computadores ainda pertencem ao mesmo grupo de comunicação após quedas de conexão. Ele utiliza duas otimizações clássicas: Compressão de Caminho e União por Rank.

void uf_inicializar(int *parent, int *rank, int n)
Estado inicial: cada nó é seu próprio "chefe supremo", sem grupos formados.
▶

Define parent[i] = i para todos os nós. Nesse ponto, existem n grupos isolados. O rank[i] = 0 indica que nenhum grupo tem profundidade ainda. Os grupos só se formam quando uf_unite é chamado.

int uf_find(int *parent, int x)
Descobre o "chefe supremo" (raiz) do grupo ao qual o nó x pertence, com compressão de caminho.
▶

Sobe na hierarquia de parent de forma recursiva até encontrar um nó que é pai de si mesmo (a raiz do grupo). Na volta da recursão, atualiza todos os nós do caminho para apontar diretamente para a raiz — isso é a compressão de caminho. Com isso, consultas futuras se tornam praticamente O(1).

c
int uf_find(int *parent, int x) {
    if (parent[x] != x)
        parent[x] = uf_find(parent, parent[x]); // compressão
    return parent[x];
}
⏱ Amortizado: O(α(n)) ≈ O(1)
void uf_unite(int *parent, int *rank, int a, int b)
Une os grupos de dois nós. O grupo de menor "rank" se torna subordinado ao maior, evitando árvores degeneradas.
▶

Encontra as raízes dos dois grupos com uf_find. Se já forem a mesma, não faz nada. Caso contrário, faz a união: o grupo com menor rank vira filho do grupo maior. Se os ranks forem iguais, qualquer um pode virar filho, mas o rank do pai é incrementado. Isso é a União por Rank.

int detectar_componentes(Grafo *g)
Usa o Union-Find para varrer todas as arestas do grafo e identificar grupos de computadores isolados.
▶

Aloca arrays de parent e rank dinamicamente. Inicializa o Union-Find e percorre todas as arestas do grafo chamando uf_unite para cada par conectado. Ao final, conta quantos nós são "raízes" do próprio grupo (uf_find(parent, i) == i): esse é o número exato de componentes conexos.

Exibe cada componente com seus membros e um alerta se a rede estiver fragmentada (num_comp > 1).

Retorno
TipoDescrição
intNúmero de componentes conexos encontrados (1 = rede íntegra, >1 = rede fragmentada)
⏱ Tempo: O(E · α(V))
